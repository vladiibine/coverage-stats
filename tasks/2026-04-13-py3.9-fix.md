# Fix coverage.py injection on Python <3.12

## Problem

On Python <3.12, coverage-stats displaces coverage.py's CTracer via `sys.settrace`. Coverage.py collects nothing — it relies entirely on data injected by `CoveragePyInterop.compute_full_arcs()` via `patch_coverage_save`.

`compute_full_arcs` uses `BranchWalker` heuristics to reconstruct arcs from line execution counts. This loses 48 arcs across 12 files on httpx compared to coverage.py standalone.

## Root cause

`compute_full_arcs` was written before the tracer recorded actual arc data. Now that `SessionStore` has real `_arc_data`, the injection should use it directly instead of heuristics.

## Fix

Update `compute_full_arcs` (or add a parallel method) to pull arcs directly from `SessionStore._arc_data` when available:

1. If `store.has_arc_data()`: iterate `store.arcs_for_file(path)`, emit every observed `(from_line, to_line)` arc. Still need entry/exit arcs and sequential arcs for line coverage in branch mode.
2. If no arc data: fall back to existing `BranchWalker` heuristic path.
3. Apply `PythonParser.translate_arcs()` on the emitted arcs before injection (same normalization coverage.py applies to its own tracer data — fixes multi-line statement mismatches).

Key files:
- `src/coverage_stats/reporters/coverage_py_interop.py` — `compute_full_arcs`, `full_arcs_for_store`
- `src/coverage_stats/store.py` — `arcs_for_file`, `has_arc_data`

## Implementation notes

### Signature change

`full_arcs_for_store(store)` calls `compute_full_arcs(path, line_data)` per file. The new arc-data path needs the store, so either:
- Pass `store` into `compute_full_arcs` as an optional kwarg, or
- Handle the arc-data path in `full_arcs_for_store` before falling back to `compute_full_arcs`

### How to use `translate_arcs`

```python
from coverage.python import PythonParser
from coverage.config import CoverageConfig

cfg = CoverageConfig()
exclude_re = "(" + ")|(".join(cfg.exclude_list) + ")"
p = PythonParser(text=source, exclude=exclude_re)
p.parse_source()
normalized = p.translate_arcs(raw_arcs)  # set of (int, int) -> set of (int, int)
```

`translate_arcs` normalizes both ends of each arc through `_multiline` (maps physical lines of multi-line statements to their canonical first line) and also applies `fix_with_jumps` (normalizes `with`-statement arcs). Guard the import — coverage.py may not be installed.

### What arcs to emit

Coverage.py in branch mode derives line coverage from arcs, so we must emit:
1. **Observed arcs from store**: `store.arcs_for_file(abs_path)` — every `(from_line, to_line)` with nonzero execution counts
2. **Entry arcs**: `(-scope_start, first_line)` for each scope — already in arc data (negative from_line)
3. **Exit arcs**: `(last_line, -scope_start)` — already in arc data (negative to_line)
4. **Sequential arcs** between consecutive executed lines in the same scope — needed so coverage.py counts every executed line, not just branch-point lines. Keep the existing sequential arc logic from `compute_full_arcs`.

## Validation

Run tests for httpx on Python 3.9, 3.10 and 3.11 with and without coverage-stats. Compare coverage.py's `covered_branches` per file — they should match.

```bash
# From coverage-stats-extensive-examples/httpx directory:

# Standalone (baseline):
.venv/bin/pytest tests --cov=httpx --cov-branch \
  --cov-report=json:/tmp/cov_39_standalone.json \
  -W "ignore::coverage.exceptions.CoverageWarning" \
  -p no:xdist -p no:coverage_stats -q --tb=no

# With coverage-stats:
.venv/bin/pytest tests --coverage-stats \
  --cov=httpx --cov-branch \
  --cov-report=json:/tmp/cov_39_with_cs.json \
  -W "ignore::coverage.exceptions.CoverageWarning" \
  -p no:xdist -q --tb=no

# Compare (run from repo root):
python3 -c "
import json
sa = json.loads(open('/tmp/cov_39_standalone.json').read())
cs = json.loads(open('/tmp/cov_39_with_cs.json').read())
for k in sorted(set(sa['files']) | set(cs['files'])):
    a = sa['files'].get(k,{}).get('summary',{}).get('covered_branches',0)
    b = cs['files'].get(k,{}).get('summary',{}).get('covered_branches',0)
    if a != b: print(f'{k.split(\"/\")[-1]}: standalone={a} with_cs={b} delta={b-a:+d}')
"
```
